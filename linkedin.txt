Just wrapped up a hands-on API penetration test against DVAPI (Damn Vulnerable API) and wrote a full industry-style vulnerability report from scratch.

Here's what I found — 5 OWASP API Top 10 (2023) vulnerabilities, all exploited manually through logic flaws alone, no automated scanners:

FIND-01 | API8 — Security Misconfiguration
The server was leaking full Node.js stack traces on JWT validation errors. Internal file paths, module names, and application logic — all handed to the attacker in a single error response.

FIND-02 | API1 — Broken Object Level Authorization (BOLA)
The /api/getNote endpoint accepted a username query parameter to fetch notes. No ownership check server-side. Changed the parameter, read the admin's private note. Classic BOLA — still one of the most common API vulnerabilities in the wild.

FIND-03 | API9 — Improper Inventory Management
The challenges endpoint used a POST body parameter to filter results. Renamed the parameter from "released" to "unreleased" and the server returned 2 hidden challenges, including sensitive data that was never meant to be public.

FIND-04 | API3 — Broken Object Property Level Authorization (Mass Assignment)
The registration endpoint didn't restrict what fields it accepted. Added "score": 9999 to the signup request body — server persisted it without question. Topped the leaderboard in one request.

FIND-05 | API5 — Broken Function Level Authorization (BFLA) [CRITICAL]
An OPTIONS request to /api/user/Charlie revealed DELETE was an allowed method — nowhere in the UI. Sent the DELETE request as a regular authenticated user. Account deleted. Flag captured. No admin role required.

All findings documented in a full pentest report with reproduction steps, impact analysis, and remediation guidance.

DVAPI is a great intentionally vulnerable lab for anyone learning API security. If you're studying for OSCP, bscp, or just getting into offensive security — highly recommend it.
